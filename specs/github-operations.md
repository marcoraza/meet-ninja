# GitHub Operations Spec

## Objetivo

Transformar decisões e tarefas de reuniões em ações rastreáveis no GitHub.

## Ações suportadas no MVP

- renderizar issue draft;
- renderizar comentário draft;
- sugerir labels;
- sugerir repo alvo;
- salvar payload local.

## Ações pós-MVP

- criar issue;
- comentar em issue ou PR;
- abrir PR com docs derivados;
- atualizar project board.

## Regras

- Nenhum write sem preview.
- Nenhum write sem autorização explícita.
- Issue deve carregar contexto suficiente para execução.
- Payload deve citar receipt ou fonte derivada.

## Formato de issue draft

- título;
- contexto;
- decisão ou problema;
- tarefas;
- critério de aceite;
- links de evidência;
- riscos.
